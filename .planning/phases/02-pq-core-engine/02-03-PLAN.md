<objective>
Implement RaBitQ as a third quantization strategy providing 32x compression with ~85%+ recall.

Purpose: RaBitQ (arXiv:2405.12497) offers 32x compression (vs PQ's 8-16x) with competitive recall. It uses randomized binary quantization with scalar correction factors, making it ideal for memory-constrained deployments. This adds RaBitQ as an independent quantization path alongside SQ8 and PQ. Implements PQ-ADV-01 (promoted from v2 scope).

Output: New `quantization/rabitq.rs` module with encode/decode/distance/training, StorageMode::RaBitQ variant.
</objective>

    1. Create `crates/velesdb-core/src/quantization/rabitq.rs` with core types:
       ```rust
       use serde::{Deserialize, Serialize};
       use crate::error::Error;

       /// Binary-quantized vector with scalar correction factors.
       #[derive(Debug, Clone, Serialize, Deserialize)]
       pub struct RaBitQVector {
           /// Binary codes packed in u64 words. Length = ceil(D / 64).
           pub bits: Vec<u64>,
           /// Correction factors: (vector_norm, quantization_inner_product).
           pub norms: (f32, f32),
       }

       /// RaBitQ index holding the random rotation matrix and dataset centroid.
       #[derive(Debug, Clone, Serialize, Deserialize)]
       pub struct RaBitQIndex {
           /// Random orthogonal rotation matrix (flattened D x D, row-major).
           pub rotation: Vec<f32>,
           /// Dataset centroid for centering.
           pub centroid: Vec<f32>,
           /// Vector dimension.
           pub dimension: usize,
       }
       ```

    2. Implement `RaBitQIndex` methods:
       - `encode(&self, vector: &[f32]) -> RaBitQVector`: Center vector (subtract centroid), compute norm, normalize, apply rotation, extract sign bits into u64 words, compute correction factors (vector_norm, quantization_inner_product between binary reconstruction and rotated normalized vector). Handle zero-norm vectors gracefully (return zero bits, norms=(0.0, 1.0)).
       - `distance(&self, query: &[f32], encoded: &RaBitQVector) -> f32`: Center query, normalize, rotate, extract sign bits, XOR+popcount against encoded bits (use `u64::count_ones()` -- compiler emits POPCNT when available), compute inner product estimate, apply affine correction, convert to L2 via `||q||^2 + ||v||^2 - 2<q,v>`.
       - `batch_distance(&self, query: &[f32], encoded: &[RaBitQVector]) -> Vec<f32>`: Process query once (center, normalize, rotate, extract bits), then iterate encoded vectors for XOR+popcount. Amortizes query preprocessing.

    3. Implement helper `signs_to_bits(values: &[f32], dim: usize) -> Vec<u64>`: Pack sign bits of f32 slice into u64 words. Bit i of word w is 1 if values[w*64 + i] >= 0.

    4. Implement helper `apply_rotation_flat(rotation: &[f32], vector: &[f32], dim: usize) -> Vec<f32>`: Matrix-vector multiply using flat row-major rotation matrix. This is shared with OPQ (Plan 04 will reuse it).

    5. Do NOT implement training in this task (Task 2 handles it). For now, `RaBitQIndex` is constructed directly in tests with known rotation matrices (identity matrix for simplicity).

       - Generate random orthogonal matrix via QR decomposition:
