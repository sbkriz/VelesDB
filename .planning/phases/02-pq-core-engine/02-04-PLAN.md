    1. Add `train_opq` function behind `#[cfg(feature = "persistence")]`:
       ```rust
       #[cfg(feature = "persistence")]
       pub fn train_opq(
           vectors: &[Vec<f32>],
           num_subspaces: usize,
           num_centroids: usize,
           opq_enabled: bool,
           opq_iterations: usize, // default 5
       ) -> Result<ProductQuantizer, Error> {
           // Validate inputs (same as train())
           // If !opq_enabled, delegate to ProductQuantizer::train() with rotation=None
           // If opq_enabled:
           //   1. Initialize rotation R = I (identity matrix)
           //   2. For each IPQ iteration:
           //      a. Apply rotation to all vectors: X_rot = X * R^T
           //      b. Train PQ on X_rot (standard k-means per subspace)
           //      c. Compute residuals for each vector (original - reconstructed)
           //      d. Solve Procrustes: find R that minimizes ||X_rot - reconstruct(quantize(X_rot))||
           //         R_new = V * U^T from SVD(X^T * Y) where Y = reconstructed
           //         Use ndarray for matrix operations
           //      e. Update R = R_new
           //   3. Final PQ training with the optimized R
           //   4. Store R as flattened Vec<f32> in ProductQuantizer.rotation
       }
       ```

    2. For the Procrustes solution without full SVD:
       - Compute cross-covariance matrix C = X^T * Y (using ndarray dot)
       - Use a simplified polar decomposition: R = C * (C^T * C)^{-1/2}
       - For the matrix square root inverse, use iterative Newton-Schulz:
         Y_0 = C^T * C / ||C^T * C||_F
         For i = 0..10: Y_{i+1} = 0.5 * Y_i * (3*I - Y_i * Y_i)
         R = C * Y_final
       - This avoids needing SVD while preserving orthogonality.
       - Alternatively, use modified Gram-Schmidt on C to extract orthogonal factor.

    3. The `apply_rotation` method (already added in Plan 02 Task 1) handles query-time rotation in `precompute_lut`. No additional changes needed for query path.

    4. Add `opq_enabled: bool` as a parameter, NOT a struct field (it's a training option, not runtime state). The runtime state is `rotation: Option<Vec<f32>>` -- if Some, rotation is applied.

    All ndarray usage inside `#[cfg(feature = "persistence")]`. Use `// TODO(EPIC-063):` for deferred optimizations.

    1. Add WGSL compute shader in `gpu/shaders.rs`:
       ```rust
       pub(super) const PQ_KMEANS_ASSIGN_SHADER: &str = r"
       struct Params {
           num_vectors: u32,
           num_centroids: u32,
           subspace_dim: u32,
           _padding: u32,
       }

       @group(0) @binding(0) var<storage, read> vectors: array<f32>;
       @group(0) @binding(1) var<storage, read> centroids: array<f32>;
       @group(0) @binding(2) var<storage, read_write> assignments: array<u32>;
       @group(0) @binding(3) var<uniform> params: Params;

       @compute @workgroup_size(256)
       fn kmeans_assign(@builtin(global_invocation_id) id: vec3<u32>) {
           let idx = id.x;
           if (idx >= params.num_vectors) { return; }

           let sd = params.subspace_dim;
           let k = params.num_centroids;
           let vec_offset = idx * sd;

           var best_dist: f32 = 3.4028235e+38; // f32::MAX
           var best_idx: u32 = 0u;

           for (var c: u32 = 0u; c < k; c = c + 1u) {
               let cent_offset = c * sd;
               var dist: f32 = 0.0;
               for (var d: u32 = 0u; d < sd; d = d + 1u) {
                   let diff = vectors[vec_offset + d] - centroids[cent_offset + d];
                   dist = dist + diff * diff;
               }
               if (dist < best_dist) {
                   best_dist = dist;
                   best_idx = c;
               }
           }
           assignments[idx] = best_idx;
       }
       ";
       ```

       ```rust
       /// Returns assignment indices (one per vector).

       #[cfg(feature = "gpu")]
       pub fn gpu_kmeans_assign(
           sub_vectors: &[Vec<f32>],
           centroids: &[Vec<f32>],
           subspace_dim: usize,
       ) -> Option<Vec<usize>> { ... }

       /// Check if GPU dispatch is worthwhile based on FLOP threshold.
       pub fn should_use_gpu(n: usize, k: usize, subspace_dim: usize) -> bool {

       }
       ```
